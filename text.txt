Описание бота для записи на маникюр
===================================

Этот бот создан для мастера маникюра и его клиентов.  
С его помощью можно полностью вести онлайн‑запись в Telegram:  
клиенты сами выбирают свободное время, а админ управляет расписанием в пару кликов.

Ниже — описание для обычных пользователей и для владельца бота (админа) простым языком.

1. Что видит и может сделать клиент
-----------------------------------

После команды `/start` клиент получает приветственное сообщение в уютном розовом стиле,  
краткое описание услуг и контакт мастера для связи.

Основные кнопки для клиента:
- «📅 Записаться» — записаться на маникюр.
- «🧾 Моя запись» — посмотреть свою текущую запись (дата и время).
- «❌ Отменить запись» — отменить свою запись.
- «💰 Прайсы» — посмотреть цены.
- «🖼 Портфолио» — перейти к примерам работ.

Также снизу (под строкой ввода) всегда есть удобные большие кнопки:
- «🏠 Меню» — вернуться в главное меню.
- «🧾 Моя запись» — быстро открыть информацию о текущей записи.

1.1. Как клиент записывается на маникюр
--------------------------------------

1. Клиент нажимает «📅 Записаться».
2. Если мастер требует обязательную подписку на канал:
   - бот просит подписаться;
   - даёт кнопку «✅ Подписаться»;
   - есть кнопка «🔁 Проверить подписку»;
   - без подписки запись недоступна.
3. Если у клиента уже есть активная запись, бот честно скажет, что сначала нужно её отменить.
4. Клиент выбирает дату в красивом календаре:
   - зелёным кружочком «🟢» помечены дни, где есть свободные окошки;
   - белым кружочком «⚪️» — дни, где записаться нельзя (нет свободных мест или вне диапазона);
   - навигация по месяцам происходит стрелочками «◀️» и «▶️».
5. После выбора даты бот показывает доступное время:
   - появляются кнопки «⏰ 11:00», «⏰ 12:30» и т.п.;
   - есть кнопка «↩️ Назад к календарю» и «🏠 В меню».
6. Бот просит ввести имя (для записи).
7. Бот просит ввести номер телефона.
8. Бот показывает итог:
   - дата;
   - время;
   - имя;
   - телефон;
   и спрашивает, подтвердить ли запись.
9. Если клиент нажимает «✅ Подтвердить»:
   - запись фиксируется;
   - клиент получает сообщение «Запись подтверждена» с датой и временем;
   - мастер получает отдельное уведомление о новой записи.

2. Запись клиента на маникюр
- При выборе «📅 Записаться»:
  - Проверка подписки на обязательный канал. Если нет:
    - Сообщение с требованием подписаться.
    - Кнопка «✅ Подписаться» (URL канала).
    - Кнопка «🔁 Проверить подписку».
    - Кнопка «🏠 В меню».
  - Проверка, есть ли у пользователя уже активная запись:
    - Если есть — предупреждение, что нужно сначала отменить текущую запись.
- Календарь для выбора даты:
  - Показывает период от сегодня до 30 дней вперёд.
  - Свободные дни, где есть хотя бы один свободный слот, помечены как «🟢ДД» и кликабельны.
  - Недоступные/закрытые или без свободных слотов — «⚪️ДД» (некликабельны).
  - Легенда над календарём: что значат 🟢 и ⚪️.
  - Навигация по месяцам стрелками ◀️ ▶️ с ограничением диапазона.
- Выбор времени:
  - После выбора даты бот получает список свободных времён из БД.
  - Строится клавиатура с вариантами «⏰ HH:MM».
  - Есть кнопка «↩️ Назад к календарю» и «🏠 В меню».
- Ввод данных клиента:
  - Имя (проверка минимальной длины).
  - Телефон (проверка минимальной длины).
- Подтверждение записи:
  - Сообщение с проверкой данных: дата, время, имя, телефон.
  - Кнопки: «✅ Подтвердить» и «❌ Отмена».
  - При подтверждении:
    - Создаётся запись в БД (с проверкой, что день открыт и слот свободен).
    - Ставится напоминание (через APScheduler) при необходимости.
    - Пользователь получает сообщение «Запись подтверждена» с данными.
    - Админ получает уведомление о новой записи:
      - ID записи.
      - ID пользователя и @username (если есть).
      - Дата, время, имя, телефон.
    - В канал расписания отправляется текстовое расписание дня (все слоты со статусами).

3. Просмотр и отмена записи клиентом
- «🧾 Моя запись»:
  - Если активная запись есть — показывает пользователю его бронь (дата и время).
  - Если нет — сообщает, что активной записи нет.
- «❌ Отменить запись»:
  - Если активной записи нет — соответствующее сообщение.
  - Если есть:
    - Запись помечается как cancelled, слот снова становится доступным.
    - Снимается напоминание.
    - Пользователю приходит сообщение: «Запись отменена. Слот снова доступен».
    - Админ получает уведомление «Запись отменена клиентом» с:
      - ID записи, ID пользователя, @username.
      - Дата, время, имя, телефон.

4. Админ-панель (inline-меню)
- Доступ только администратору (по admin_id).
- Пункты меню:
  - «➕ Добавить рабочий день»
  - «🚫 Закрыть день полностью»
  - «➕ Добавить слот»
  - «➖ Удалить слот»
  - «📋 Расписание на дату»
  - «📅 Записи на неделю»
  - «📆 Рабочие дни»
  - «🔁 Перенести запись»
  - «🗑 Отменить запись (по ID)»

5. Управление рабочими днями и слотами (админ)
- Добавить рабочий день:
  - Выбор даты через админ-календарь (или ввод YYYY-MM-DD).
  - В админ-календаре:
    - 🟢ДД — день открыт (есть запись в work_days с is_open=1).
    - ⚪️ДД — день закрыт.
  - При выборе/вводе даты она добавляется в work_days как открытая.
- Закрыть день полностью:
  - Выбор даты как выше.
  - День помечается закрытым, все слоты деактивируются.
  - Все активные записи этого дня отменяются:
    - Для каждой снимается напоминание.
    - Каждому клиенту отправляется уведомление об отмене (дата/время).
  - Админу показывается, сколько записей отменено.
- Добавить слот:
  - Выбор даты (календарь/ввод).
  - Выбор времени:
    - Ввод HH:MM или выбор из подсказок каждые 30 минут (09:00–20:30).
  - В БД:
    - День автоматически добавляется и открывается, если его не было.
    - В таблицу slots добавляется/активируется слот.
- Удалить слот:
  - Выбор даты и времени аналогично «Добавить слот».
  - Слот помечается неактивным.
  - Если на нём была активная запись:
    - Запись отменяется.
    - Напоминание снимается.
    - Клиенту отправляется сообщение, что запись отменена админом.

6. Просмотр расписания и рабочих дней (админ)
- «📋 Расписание на дату»:
  - Выбор даты.
  - Бот показывает список всех слотов этого дня с пометками:
    - ✅ свободно
    - ⛔️ занято
    - 🚫 слот выключен
- «📆 Рабочие дни»:
  - Показывает список открытых дней только на ближайшие 2 недели (от сегодня +14 дней).
- «📅 Записи на неделю»:
  - Для текущей недели (понедельник–воскресенье) выводит все активные записи:
    - По дням.
    - Для каждой: время, имя, телефон, user_id, @username, ID записи.

7. Отмена записи админом по ID
- Кнопка «🗑 Отменить запись (по ID)»:
  - Бот выводит список активных записей (до 50 штук) с подробной информацией и ID.
  - Под списком — кнопки «ID N — Дата Время» для быстрого выбора.
  - Также можно ввести ID записью вручную.
- При отмене:
  - Запись помечается как cancelled, снимается напоминание.
  - Клиенту отправляется сообщение:
    - «Ваша запись отменена администратором» с датой и временем.
  - Админ получает подтверждение, что запись отменена и слот снова доступен.

8. Перенос записи администратором
- Кнопка «🔁 Перенести запись»:
  - Шаг 1: выбор записи
    - Показ списка активных записей (аналогично отмене по ID).
    - Можно:
      - нажать кнопку с нужным ID,
      - или ввести ID вручную (бот переключается в режим переноса для этой записи).
  - Шаг 2: выбор новой даты
    - Бот показывает админ-календарь:
      - 🟢 — открытые дни.
      - ⚪️ — закрытые дни.
    - Можно выбирать через календарь или ввести дату вручную.
  - Шаг 3: выбор нового времени
    - Ввод HH:MM или выбор из подсказок (каждые 30 минут).
    - Бот проверяет:
      - что день открыт,
      - что новый слот активен и свободен.
    - Если всё ок:
      - Обновляет day/time записи.
      - Пересоздаёт напоминание для новой даты/времени.
      - Клиенту отправляет уведомление:
        - «Ваша запись перенесена администратором» с новой датой и временем.
      - Админ получает подтверждение с новой датой и временем.

9. База данных и напоминания
- SQLite структура:
  - users: информация о пользователях Telegram.
  - work_days: список рабочих дней и их статус (открыт/закрыт).
  - slots: слоты времени по дням, с флагом активности.
  - bookings: записи клиентов (активные/отменённые), с полями:
    - user_id, day, time, name, phone, status, created_at, cancelled_at.
  - reminders: напоминания, связанные с booking_id (для APScheduler).
- Логика:
  - Только один активный booking на (day,time).
  - Пользователь не может иметь более одной активной записи одновременно.
  - При старте бота все будущие активные записи восстанавливают свои напоминания.
